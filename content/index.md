# WarpFusion

## 面向安全分析场景的实时关联计算引擎

WarpFusion 专注于安全事件关联、风险告警归并与实体行为分析。  
它不是通用流计算 SQL 平台，也不是任意 DAG 引擎，而是围绕 `.wfs`、`.wfl`、`fusion.toml` 三文件模型构建的专用运行时与工具链。

### 关键特性

#### 1. WFL 面向实时关联与风险检测

WFL 不是通用查询语言，而是面向检测规则的专用 DSL。它的核心表达能力可以拆成 5 个通栏能力：

- 事件绑定 `events`
  作用：把输入 window 和过滤条件绑定成规则里的事件别名  
  特点：先把“看什么事件”定义清楚，再进入后续匹配和判断

- 时序匹配 `match<key:duration>`
  作用：描述按实体、按时间窗口的关联检测  
  特点：适合表达“先发生 A，再发生 B”“5 分钟内累计达到阈值”

- 缺失与关闭语义 `on close`
  作用：写 timeout、flush、eos 阶段的判断  
  特点：不仅能检测“发生了什么”，也能检测“本该发生但没有发生什么”

- 关联与归属 `join + entity`
  作用：关联外部窗口，并明确告警最终属于谁  
  特点：把实时事件、外部上下文和归属实体放进同一条规则链

- 输出结果 `score + yield`
  作用：给规则评分，并产出中间结果或最终告警  
  特点：规则不只负责命中判断，还能直接定义输出结构和下游消费方式

一句话说，WFL 的核心表达能力，就是把事件绑定、时序匹配、缺失检测、外部关联和告警输出收敛为一套面向实时检测的规则语言。

#### 2. 同时支持实时接入与离线回放

运行时支持两种模式：

- `daemon`：在线常驻运行，接收实时数据
- `batch`：离线文件回放，用于验证、复盘和测试

输入源支持：

- TCP source
- file source

文件输入支持：

- `ndjson`
- `arrow_framed`
- `arrow_ipc`

这让线上处理、离线验证、测试回放能够共享同一套规则语义。

#### 3. 规则、配置、数据生成形成工具闭环

WarpFusion 不只是运行时，还提供围绕规则开发的完整工具链：

- `wfusion config render/origins/diff/vars`
  用于查看配置 merge、变量展开与来源
- `wfl explain/lint/fmt/replay/test/verify`
  用于规则解释、检查、格式化、回放与契约验证
- `tree-sitter-wfl`
  提供 WFL 语法高亮支持，便于编辑器集成、规则编写和本地阅读体验
- `wfgen gen/lint`
  用于测试数据生成、一致性校验和发送验证

这套工具链让规则开发从“写规则”扩展到“解释、验证、回放、比对”。

#### 4. Window-Centric 运行时模型

WarpFusion 的核心不是“消息直接推给规则”，而是以 Window 为中心组织计算：

`Receiver -> Router -> Window -> Rule Engine -> Alert Sink`

运行时关键机制包括：

- 基于事件时间的 watermark
- `allowed_lateness` 与迟到策略
- 基于时间和内存的 eviction
- 每个 rule 独立 task
- 基于 cursor 的增量读取
- 基于 `Notify` 的通知驱动执行

这样可以让窗口语义真正参与规则执行，而不是停留在存储层。

#### 5. 支持逐条 enrichment 与下游聚合

除了窗口匹配，WarpFusion 还支持 `on each` 模式：

- 对单条事件做 enrichment、语义整理或风险打分
- 把中间结果写入下游 window
- 在下游规则里继续做窗口聚合和关联

这让规则链路既能表达单条事件语义，也能表达后续时序检测。

#### 6. 三文件模型清晰分层

WarpFusion 把系统拆成三类文件，职责边界明确：

- `.wfs`：定义逻辑数据窗口、字段、时间列和保留时长
- `.wfl`：定义检测逻辑，包括 `events`、`match`、`join`、`entity`、`yield`
- `fusion.toml`：定义运行模式、sources、runtime、window 参数和 sinks

规则逻辑与运行参数分离，可以降低配置耦合，提升可维护性。

#### 7. 面向关联检测，而不是通用流平台

WarpFusion 的能力边界是明确的：

- 面向多事件、多窗口、多实体的实时检测
- 面向缺失检测、时序链、阈值判断、实体归并
- 面向规则可解释、可调试、可验证的工程化落地

这种设计让系统更容易控制，也更适合安全分析场景长期演进。

#### 8. 轻量、单机、无外部依赖

WarpFusion 的定位不是依赖 Kafka、数据库、集群控制面的重型平台，而是：

- 单机可运行
- 资源占用可控
- 结构清晰
- 易于调试与本地验证

这使它更适合规则研发、实验场景、单机场景和早期生产部署。

### 适合什么场景

WarpFusion 特别适合以下问题：

- 5 分钟内同一源 IP 登录失败 3 次后又出现扫描行为
- 某类请求已发出，但在预期时间内没有收到响应
- 先对单条事件做语义打分，再按实体或窗口聚合风险
- 在离线样本上复盘规则行为，并与线上规则保持一致

### 一句话总结

WarpFusion 是一个以 WFL 为核心、以 Window 为中心、同时覆盖实时接入、离线回放、规则验证与配置诊断的轻量实时关联计算引擎。
